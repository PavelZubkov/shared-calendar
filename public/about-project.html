ShaCal - about the project

Summary

Shacal allows users to publicly share their google calendars and it's events. Any visitor can view an event details and add his email to the attendees list without event's owner taking any actions.

Auth with Google

Shacal requires following permissions to provide a full functionality:

  • Associate you with your personal info on Google (account ID)
  • View your email address
  • See the list of Google calendars youâ€™re subscribed to
  • View and edit events on all your calendars

Permission: account ID

Shacal stores users' google account IDs. The account ID is used to identify data that belong to user.

Permission: email

Shacal stores users' email address. The email address is used to check if user has permissions to add new events into calendars owned by other users.

In addition the email address can be used to send user an email with important information.

Permission: view calendars list

Shacal shows users a list of calendars where user has a write permission. User can choose any of listed calendars and create a link for public anonymous access to the list of the calendar's events ("share" button). Shacal stores only the calendar's ID to relate public link to shared calendar.

Permission: view and edit events

Shacal provides a link for public anonymous view of any following event from a calendar shared by user. An anonymous user can add his email (after passing the confirmation of ownership procedure) to the list of the event's attendees. After that anonymous user will receive an email from Google Calendar that calendar's owner have invited him to the event.

Shacal provides UI to create new events in a shared calendar. The calendar's owner can allow other users to create new events in the calendar. The calendar's owner can edit any event in the calendar. Other users can edit authored events.

Shacal does not store any event information except event ID and related calendar ID.
